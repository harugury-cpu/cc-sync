# kkirikkiri (끼리끼리)

> **한마디면 충분. AI 에이전트 팀이 알아서 모인다.**

하고 싶은 걸 자연어로 말하면, 끼리끼리가 2–3개 질문으로 인터뷰하고, 환경을 스캔하고, 팀을 제안한 뒤 실행합니다. Claude Code 안에서 전부 이루어집니다.

## 빠른 시작

### 1. 마켓플레이스 추가

```
/plugin marketplace add https://github.com/fivetaku/gptaku_plugins.git
```

### 2. 설치

```
/plugin install kkirikkiri
```

### 3. Agent Teams 활성화

```json
// ~/.claude/settings.json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

### 4. 실행

```
/kkirikkiri 리서치 팀 만들어줘
```

## 왜 끼리끼리인가?

- **자연어 입력 → 실행 중인 팀** — YAML도, 에이전트 정의 파일도 직접 작성할 필요 없음
- **인터뷰 기반** — 긴 설정 폼 대신 2–3개의 핵심 질문으로 팀을 설계함
- **환경 인식** — 설치된 도구(Codex CLI, Gemini CLI, `.claude/agents/`)를 탐지해 실제 가용 자원으로 최적 팀을 구성함
- **멀티 모델** — Claude, Codex CLI, Gemini CLI가 하나의 팀 안에서 각기 다른 역할을 맡을 수 있음
- **검증 루프** — 1라운드 결과가 부족하면 팀을 자동으로 재시도하거나 재구성함 (최대 3라운드)
- **공유 메모리** — `.kkirikkiri/` 파일이 라운드 간에 유지되어 교체된 팀원도 바로 맥락을 파악할 수 있음
- **에이전트 재사용** — 잘 동작한 팀원을 `.claude/agents/`에 저장해 다른 프로젝트에서도 쓸 수 있음

이름의 유래는 한국어 관용어 **끼리끼리** — *비슷한 것들이 자연스럽게 모인다*는 뜻. 팀은 항상 공통 목적을 중심으로 모입니다.

## 어떻게 작동하나?

```
자연어 입력
    → Step 1: 의도 파악 + 프리셋 매칭
    → Step 2: 환경 스캔 (병렬)
    → Step 3: 인터뷰 — 2–3개 AskUserQuestion
    → Step 4: 동적 팀 구성
    → Step 5: 팀 구성 제안 + 사용자 확인
    → Step 6: 공유 메모리 초기화 + 팀 실행
    → Step 7: 품질 검증 루프 (최대 3라운드)
    → Step 8: 결과 수집 + 리포트
```

**팀장 원칙:**
- 팀장은 항상 가장 성능 좋은 모델 (기본값: Opus)
- 팀장은 계획, 배분, 검증만 — 코드를 직접 작성하지 않음
- 각 팀원의 역할은 엄격히 분리됨

## 기능

### 프리셋

자연어 키워드 매칭으로 자동 선택되는 5종 프리셋:

| 프리셋 | 트리거 키워드 | 기본 구성 |
|--------|------------|----------|
| 리서치 | 조사, 리서치, 찾아줘, 비교해줘 | 팀장 + 리서처 2명 |
| 개발 | 만들어줘, 구현해줘, 코딩해줘 | 팀장 + 개발자 2명 |
| 분석 | 분석해줘, 리뷰해줘, 검토해줘 | 팀장 + 탐색자 2명 |
| 콘텐츠 | 써줘, 문서, README, 블로그 | 팀장 + 작성자 + 검토자 |
| PM/제품 | PRD, 전략, 기획, OKR, GTM | 팀장 + PM + 리서처 |

프리셋은 출발점입니다. 인터뷰와 환경 스캔에 따라 매번 다른 팀이 구성됩니다.

### 공유 메모리

팀이 프로젝트 루트의 `.kkirikkiri/` 디렉토리에 작업 내용을 기록합니다:

| 파일 | 역할 |
|------|------|
| `TEAM_PLAN.md` | 작업 계획, 역할 배분, 목표 |
| `TEAM_PROGRESS.md` | 완료/미완료 진행 현황 |
| `TEAM_FINDINGS.md` | 발견 사항, 실패한 접근(`DEAD_ENDS`) |

팀원이 교체되더라도 이 파일들이 남아 있어 새 팀원이 즉시 맥락을 이어받을 수 있습니다.

### 검증 루프

| 라운드 | 전략 |
|--------|------|
| 1라운드 | 원래 팀이 작업 실행 |
| 2라운드 | 자동 판정: 유지(A) / 전면 교체(B) / 부분 교체(C) |
| 3라운드 | 무조건 새 팀으로 전면 재구성 |

### 멀티 모델 지원

Claude + Codex CLI + Gemini CLI가 하나의 팀에서 각각 다른 역할을 맡을 수 있습니다. 설치된 도구를 자동으로 감지해 최적화하고, 외부 CLI가 없어도 Claude만으로 전체 팀을 구성합니다.

### 에이전트 자동 탐지 및 재사용

`.claude/agents/`에 에이전트 정의가 있으면, 끼리끼리가 자동으로 감지하고 프리셋에 맞는 에이전트를 우선 추천합니다:

| 프리셋 | 추천 에이전트 예시 |
|--------|-----------------|
| 리서치 | deep-research, data-analyst |
| 개발 | code-reviewer, architect |
| 분석 | code-analyzer, security-reviewer |
| 콘텐츠 | writer, translator |

작업 완료 후 잘 동작한 팀원을 `.claude/agents/`에 저장하면 다른 프로젝트에서도 재사용할 수 있습니다.

### 스폰 안정성

팀원이 합류에 실패하면 자동으로 재시도합니다:
1. 동일 설정으로 1회 재시도
2. 모델 다운그레이드 후 재시도
3. 나머지 팀원으로 계속 진행

### 팀 저장 및 불러오기

```
/kkirikkiri 저번 리서치 팀 다시 써줘
```

## 요구사항

### 필수

- **Claude Code** (최신 버전)
- **Agent Teams 활성화:**
  ```json
  // ~/.claude/settings.json
  {
    "env": {
      "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
    }
  }
  ```
- **tmux:** `brew install tmux` (macOS) / `apt install tmux` (Linux)
- **Node.js** (외부 CLI 연동용)

### 선택 (멀티 모델)

```bash
npm install -g @openai/codex       # Codex CLI (코드 분석/리뷰)
npm install -g @google/gemini-cli  # Gemini CLI (디자인/대규모 분석)
```

없어도 동작합니다. Claude만으로 팀을 구성합니다.

### 비용 참고

| 팀 규모 | 예상 시간 | 비용 수준 |
|---------|----------|----------|
| 2–3명 | 5–15분 | 낮음 |
| 4–5명 | 10–30분 | 중간 |
| 5명+, 멀티라운드 | 30분–1시간 | 높음 |

팀원 수를 줄이거나 Codex/Gemini CLI를 활용하면 비용을 절약할 수 있습니다.
