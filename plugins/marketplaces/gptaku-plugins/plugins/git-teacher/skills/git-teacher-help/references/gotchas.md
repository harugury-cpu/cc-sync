# Git 주의사항 (Gotchas)

모든 스킬에서 참조하는 공유 주의사항 목록. 비개발자가 자주 겪는 함정과 혼란을 클라우드 비유로 설명한다.

## Git 조작 시 주의사항

### Push가 거절될 때

**상황**: `git push`가 `rejected (non-fast-forward)` 에러를 보여줄 때

**설명 방법**: "상대방이 먼저 올린 게 있어요. 최신 내용을 먼저 받아온 다음에 다시 올릴게요."

> "push 실패"라고 하면 비개발자가 당황한다. "실패"가 아니라 "순서를 맞추는 과정"임을 강조한다.

**대응**: `git pull`을 먼저 안내하고, 충돌이 있으면 충돌 해결 플로우로 진행한다.

### .gitignore에 추가해도 파일이 계속 보일 때

**상황**: `.gitignore`에 파일을 추가했는데 Git이 여전히 추적하는 경우

**설명 방법**: ".gitignore는 '앞으로 새로 들어오는 파일'만 무시해요. 이미 클라우드 폴더에 올라간 파일은 따로 '추적 해제'를 해줘야 해요."

> `git rm --cached`가 필요하지만, "삭제"라는 단어에 비개발자가 겁먹는다. "추적 해제"로 설명하고, 실제 파일은 그대로 남아 있다는 점을 반드시 안심시킨다.

**대응**: "파일은 그대로 있고, Git한테 '이제 이 파일은 신경 쓰지 마' 하고 알려주는 거예요."

### 빈 폴더가 Git에 안 올라갈 때

**상황**: 빈 폴더를 만들었는데 `git status`에 표시되지 않는 경우

**설명 방법**: "Git은 파일만 기억해요. 빈 폴더는 기억 못 해서, 폴더를 유지하고 싶으면 안에 표시를 남겨둬요."

> 관례적으로 `.gitkeep`이라는 빈 파일을 넣어둔다. "빈 폴더에 메모지를 한 장 넣어두는 거예요."

**대응**: `.gitkeep` 파일을 자동 생성하고 안내한다.

### GitHub 100MB 파일 제한

**상황**: 100MB 이상 파일을 push하면 에러가 발생하는 경우

**설명 방법**: "클라우드 폴더에 너무 큰 파일은 못 올려요. GitHub은 파일 하나당 100MB까지만 허용해요."

**대응**: 해당 파일을 `.gitignore`에 추가하도록 안내한다. "이 파일은 '무시 목록'에 넣어서 올리지 않게 할게요."

## 비개발자 대응 시 주의사항

### Merge Conflict 용어

**절대 금지**: "HEAD", "incoming change", "current change" 용어 사용 금지

**대신 사용**:
- HEAD / current change → **"내가 작업한 부분"**
- incoming change → **"상대방이 작업한 부분"**

**설명 방법**: "두 사람이 같은 부분을 동시에 고쳐서, 어느 쪽을 쓸지 골라야 해요."

> 기술 용어 없이 "내 것" vs "상대방 것"으로만 선택지를 제시한다.

### Git Stash 설명

**비유**: "임시 보관함"

| Git 명령 | 비유 | 설명 |
|----------|------|------|
| `git stash` | 임시 보관함에 넣기 | "작업 중인 내용을 잠깐 치워두는 거예요" |
| `git stash pop` | 꺼내기 (보관함에서 사라짐) | "보관함에서 꺼내면 보관함은 비워져요" |
| `git stash apply` | 복사하기 (보관함에 남아있음) | "보관함에서 복사해오는 거라 원본은 그대로예요" |

**설명 방법**: "지금 하던 작업을 잠깐 서랍에 넣어두고, 다른 일을 먼저 할 수 있어요. 끝나면 서랍에서 다시 꺼내면 돼요."
