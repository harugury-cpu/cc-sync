인디고 보라와 그라디언트 글로우

Stripe의 마케팅 사이트는 "금융 인프라를 디자인 언어로 번역한 사이트"다. 대부분의 결제 SaaS가 파란색과 안전감으로 도배할 때, Stripe는 오히려 인디고 보라 #635BFF와 그라디언트 글로우로 "기술이 발전하는 방향"을 시각화한다. hero 섹션 배경은 순백 #FFFFFF이지만, 텍스트 제목 뒤로 #BDB4FF → #643AFD → #533AFD 가로 그라디언트가 조용히 흐른다.

색상 전략은 "brand indigo 한 축 + neutral blue-gray 두 축"이다. 브랜드는 #635BFF와 #533AFD 한 쌍으로 압축되고, neutral은 light 쪽 #F8FAFD / #E5EDF5 / #D4DEE9 + dark 쪽 #1A2C44 / #11273E / #0D253D / #061B31의 두 계단을 평행 운용한다. 그래서 light 섹션과 dark 섹션이 같은 페이지에서 교차해도 어색하지 않다.

이 사이트의 진짜 시그니처는 shadow다. --hds-color-shadow-xs/sm/md/lg/xl 각 레벨이 top + bottom 2-layer 원자로 정의되어 있어서, 모든 카드가 "위에서 내려오는 빛"과 "아래로 떨어지는 그림자"를 동시에 받는다. Linear의 5-layer stack과는 완전히 다른 설계 철학.

타이포그래피는 자체 호스팅한 sohne-var이 축이다. variable axis로 400/425/450/500/550/600/700의 미세 weight를 제공하며, 본문은 16px weight 400, 헤딩은 500~600 variable axis 값을 정확히 사용. 코드 폰트는 SourceCodePro. HDS 네임스페이스 아래 --hds-font-heading-* 8단 반응형 scale이 선언되어 있다.

레이아웃은 10단계 breakpoint로 촘촘하다 — 400/480/600/640/768/840/940/1115/1264/1300px. 버튼 padding까지 반응형 3단 성장으로 설계되어 화면이 커지면 CTA가 자체 scale up된다. 인터랙션은 scroll-linked carousel / bento grid가 중심이다.

Stripe — --hds-*

URL 하나로 추출한 실제 CSS 토큰 기반 디자인 시스템 리포트. hard-coded 색·폰트·여백이 아니라 라이브 사이트의 값 그대로.

/* 1. 폰트 */
body {
  font-family: "sohne-var",
    "SF Pro Display",
    -apple-system, sans-serif;
  font-weight: 400;
  font-size: 16px;
  line-height: 1.5;
}
/* 2. 배경 + 텍스트 */
:root {
  --bg: #FFFFFF;
  --fg: #0D1738;
  --surface: #F8FAFD;
  --border: #D4DEE9;
  --text-muted: #64748D;
}
body {
  background: var(--bg);
  color: var(--fg);
}
/* 3. 인디고 + gradient */
:root {
  --brand: #635BFF;
  --brand-strong: #533AFD;
  --accent-gradient:
    linear-gradient(90deg,
      #BDB4FF, #643AFD, #533AFD);
}
절대 하지 말 것: 브랜드 #635BFF를 flat 배경으로 쓰지 마라. Stripe의 인디고는 CTA, focus ring, gradient stop에만 등장한다. Flat indigo 배경은 Discord 또는 Linear인데, Stripe가 아니다. Stripe에서 indigo를 면적으로 쓰는 유일한 방법은 gradient다.

Do / Don't

실제 CSS에서 검증된 규칙만 수록.

DO

  • 배경은 #FFFFFF(light) 또는 #0D253D(dark). 같은 페이지에서 교차 가능.
  • CTA primary: #635BFF 배경 + #FFFFFF 텍스트 + border-radius: 8px.
  • 본문은 sohne-var 16px weight 400 color #3C4F69 또는 #0D1738.
  • Card radius는 12px 기본, 16px hero.
  • Shadow는 반드시 top + bottom 2-layer dual.
  • Heading weight: 500 / 550 / 600 variable axis 값 사용.
  • Stat number 색상 순환: #0810BF → #4304EA → #8A35DF → #C42FA5.
  • Gradient는 #BDB4FF → #643AFD → #533AFD 3-stop 90deg 수평.

DON'T

  • #635BFF를 flat 배경으로 쓰지 말 것 — indigo는 CTA / focus / gradient stop에만.
  • 본문을 #000000로 두지 말 것 — 대신 #0D1738 (heading) 또는 #3C4F69 (body).
  • 단층 box-shadow 금지 — 반드시 2-layer dual.
  • body에 font-weight: 500을 쓰지 말 것 — body는 400. Heading만 500/550/600.
  • radius 10px 같은 비표준 값 금지 — 4 / 8 / 12 / 16 / 20 중 하나.
  • Single breakpoint 반응형 금지 — Stripe는 10단계 + heading size 독립 성장.
  • sohne-var 대신 non-variable Söhne 금지 — weight 550/425 잃는다.
  • transition 400ms 이상 금지 (scroll animation 제외).
  • Gradient 수직(180deg) 금지 — 90deg 수평 고정.
  • --hds-color-brand-100을 CTA 배경에 쓰지 말 것 — 500/600만 primary.
