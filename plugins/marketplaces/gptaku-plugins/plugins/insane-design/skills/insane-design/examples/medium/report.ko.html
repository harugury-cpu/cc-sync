Design System Report

Medium — 실제 live CSS 기준

현재 접근 가능한 Medium의 핵심은 warm hero `#F7F4ED`, GT Super headline, Sohne UI, 그리고 `#1A8917` CTA다. 요청받은 legacy dark / yellow reference는 이번 live CSS 수집 범위에서 재확인되지 않아 canonical token에서 제외했다.

Quick Start 5분 안에 Medium처럼 만들기
1 · UI 폰트 body { font-family: "sohne", "Helvetica Neue", Helvetica, Arial, sans-serif; font-weight: 400; color: #242424; }
2 · hero 톤 .hero { background: #F7F4ED; border-bottom: 1px solid #242424; } .hero-title { font-family: "gt-super", Georgia, serif; }
3 · CTA 축 .btn-primary { background: #1A8917; border-color: #1A8917; color: #FFFFFF; border-radius: 99em; } .btn-primary:hover { background: #156D12; }
절대 하지 말아야 할 것 하나: hero headline까지 전부 sans-serif로 밀어붙이지 마라. 현재 Medium의 첫인상은 gt-super 한 방에서 나온다.

09 · Layout Patterns

레이아웃 패턴

headline, subtitle, pill, image 네 가지 재료를 route마다 재배치하는 구조다.

landing hero

  • 배경 #F7F4ED
  • headline column max-width 720px
  • image 460x600 우측 배치
  • rail 75px height + 1px #242424 border

topic page

  • page title 42px / 52px
  • feed card title 20px / 24px
  • meta 13px / 20px + #6B6B6B
  • surface white + minimal decoration

secondary pages

  • /about는 white surface 비중이 더 높다
  • sign-in flow는 compact CTA 중심 sheet다
  • custom property보다 route-specific class sheet에 의존한다

depth strategy

  • 그림자보다 경계선을 우선한다
  • 면 분리는 #F7F4ED / #FFFFFF 톤 차이로 해결한다
  • 강조는 green CTA와 one-shot display serif로 끝낸다

11 · Content Voice

라이팅 원칙

현재 landing copy는 짧고 조용하며, 행동을 한 단어 CTA로 끝낸다.

플랫폼 헤드라인
추상 명사 2~3개로 identity를 압축한다.
“Human stories & ideas”
서브카피
동사 나열로 효용을 설명하고, 문장은 짧게 끝낸다.
“A place to read, write, and deepen your understanding”
CTA
완전한 문장 대신 1~2단어 action verb로 끝낸다.
“Get started” / “Start reading” / “Sign in”
메타 정보
작성자보다 읽기 맥락을 먼저 보조한다.

13 · Do / Don't

Do / Don't

Medium 디자인에서 지킬 것과 피할 것.

✓ DO

  • hero는 GT Super로 한 번만 크게 튀게 만든다
  • UI는 Sohne로 일관되게 유지한다
  • `#1A8917`는 primary CTA에만 쓴다
  • 배경 분리는 `#F7F4ED`와 `#FFFFFF`의 면 차이로 해결한다
  • flat surface + 1px border를 우선한다

✗ DON'T

  • headline까지 sans로 처리하지 않는다
  • green wash로 전체 페이지를 덮지 않는다
  • 그림자 카드 UI를 얹지 않는다
  • 임의의 custom property palette를 live CSS인 것처럼 꾸미지 않는다
  • 검증되지 않은 legacy dark / yellow refs를 current canonical token으로 쓰지 않는다
