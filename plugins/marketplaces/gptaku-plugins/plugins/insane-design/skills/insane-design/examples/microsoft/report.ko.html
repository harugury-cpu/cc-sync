Overview

Microsoft 홈은 Fluent 단일체가 아니라 Reimagine + UHF 하이브리드다

본문은 `reimagine-*` 웹 컴포넌트와 `--ds-*` 토큰으로 돌아가고, chrome은 여전히 `#0067B8` 중심의 UHF CSS를 쓴다. 이 두 축을 같이 읽어야 live Microsoft가 보인다.

Modular Fluentlegacy blue chrome + Fluent-like semantic alias

live CSS에서 공개 Fluent Web 변수명 `--colorBrandBackground`는 나오지 않는다. 대신 Reimagine bundle이 `--ds-theme-background-accent-strong-normal` 같은 내부 alias를 쓰고, global chrome은 `#0067B8` literal을 유지한다.

AEM / Cascade Reimagine UHF chrome Segoe UI Variable hero carousel 7000ms

핵심 결론

  • `store...min.css`가 본문 Reimagine 토큰 `--ds-*` 832개를 정의하고, page 전체 기준 `--ds-*`는 842개까지 확장된다.
  • `base...min.css`와 UHF CSS는 링크 / cart / chrome에 `#0067B8`를 유지한다.
  • Segoe UI Variable 3패밀리와 Segoe UI fallback이 동시에 선언된다.
Fluent token audit 공개 Fluent Web 이름(`--colorBrandBackground`, `--colorNeutralForeground1`)은 live CSS에서 직접 발견되지 않았다. 실제 페이지는 같은 의미층을 `--ds-theme-*` alias로 노출한다.

Layout Observations

  • hero는 full-bleed visual이라기보다 editorial carousel이다.
  • card 영역은 `` + 16:9 media로 정리된다.
  • chrome은 `uhf-header`, `uhf-footer-nav-group`, cart/search/mecontrol 같은 shell element가 담당한다.
  • dark night variant는 hero나 일부 featured module에서만 사용되고, 전체 root theme는 light base를 유지한다.

Motion Observations

  • hero carousel autoplay는 7000ms로 길고 조용하다.
  • hover/fill/background transition은 거의 전부 `.2s`다.
  • media reveal만 `opacity .3s ease-in-out`로 조금 더 길다.
  • 과장된 scroll FX보다 상태 변화의 안정감이 우선이다.

Rules

Microsoft답게 보이게 하는 최소 규칙

두 블루를 역할별로 분리하고, Segoe UI 계열과 module-based structure를 유지하는 것이 가장 중요하다.

DO

  • `#0067B8`는 link / header / chrome accent로 사용한다.
  • `#0078D4`는 tokenized accent fill로 쓰고 hover는 `#006DC1`로 내린다.
  • Segoe UI Variable / Segoe UI 계열을 유지한다.
  • hero와 featured card는 16:9 media 중심으로 설계한다.
  • spacing은 2 / 4 / 8 / 12 / 16 / 24 / 32 / 48 / 56px 계단을 따른다.
  • hover는 짧게, hero 이동은 carousel rhythm으로 처리한다.

DON'T

  • `#0067B8`와 `#0078D4`를 하나의 primary color처럼 합치지 말 것.
  • Fluent 공개 토큰명(`--colorBrandBackground`)이 live page에 있다고 가정하지 말 것.
  • Inter, Roboto 같은 generic SaaS 폰트로 덮어쓰지 말 것.
  • 모든 section을 동일한 white card grid로만 반복하지 말 것.
  • 느린 hover / 과장된 scroll FX를 넣지 말 것.
  • radius를 단일 12px로 고정하지 말 것. live page는 4 / 8 / 10 / 16 / 24 / 25px literal을 섞는다.
