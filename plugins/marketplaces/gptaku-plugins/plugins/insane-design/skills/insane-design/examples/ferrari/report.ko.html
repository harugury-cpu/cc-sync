Dark cinema with a single red accent.

Ferrari의 웹 경험은 `#181818` 근-검정 위에 자동차를 전시하는 방식으로 럭셔리와 속도를 동시에 밀어낸다. 이 리포트는 기존 placeholder 기반 초안을 버리고, `design.md`에 명시된 실제 Ferrari 토큰과 규칙만으로 재구성했다.

레드를 배경 전체에 쓰지 않는다. Ferrari는 레드를 섹션 바탕색이 아니라 그라디언트 버튼과 accent line에만 올린다.
01 · Provenance

출처와 토큰 범위

아래 정보는 `design.md`에 기록된 Ferrari CSS 수집 결과와 서술을 그대로 반영한다. 이 보고서에는 빈 표기나 설명용 placeholder를 넣지 않고, 확인된 값만 남겼다.

핵심 해석: Ferrari는 밝은 배경에 레드를 칠하는 방식이 아니라 `#181818` 다크 스테이지 + `#DA291C` 한 점 accent로 긴장감을 만든다. 회색 ramp와 노란 포커스가 실제 UI 계층을 정리하고, 라이트 섹션은 `--f-color-background-0`와 `--f-color-background-10`만 제한적으로 사용한다.
02 · Typography

FerrariSans 단일 시스템

`design.md` 기준 Ferrari의 기본 스택은 `FerrariSans` 단일 시스템이며, 폴백은 `Helvetica Neue`다. 아래 프리뷰는 실제 CSS 스택 그대로 렌더링하므로, 로컬에 FerrariSans가 없으면 Helvetica Neue로 떨어진다.

Font Preview
Ferrari F80 // Maranello
Display 28px에서 Micro 9px까지 동일한 가족을 유지하고, 헤딩은 1.23 line-height로 긴장감을 만든다.
Font Stack
`"FerrariSans"`
`"Helvetica Neue"`, Helvetica, Arial, sans-serif
CSS 변수는 `var(--f-ferrari-font)`이며, 기본 weight는 `400`. 한국어 설명은 Pretendard Variable로 읽기 좋게 보강하고, 실제 프리뷰는 Ferrari 스택으로 출력한다.
03 · Color

Dark ramp, controlled red, sharp focus yellow

클릭 가능한 스와치는 hex 값을 클립보드로 복사한다. alias 토큰은 원문 값을 그대로 보여주고, 실제 렌더링에 쓰이는 hex도 함께 제공한다.

04 · Spacing

4px에서 128px까지의 명확한 스케일

Ferrari spacing은 `--f-space-xxxs`부터 `--f-space-super`까지 명확하게 커진다. 작은 UI 간격은 4/8/16px에서 시작하고, 큰 레이아웃 전환은 64/96/128px에서 만들어진다.

레이아웃 감각: 전체 viewport hero, 다크/라이트 섹션 교차, 그리고 좌측 혹은 하단 오버레이 배치가 Ferrari의 기본 구문이다.
05 · Effects

Pill radius, zero-radius cards, restrained shadow

Ferrari는 중간 radius를 거의 쓰지 않는다. 버튼은 `9999px`, 카드와 이미지 컨테이너는 `0`. 그림자도 과장하지 않고 `0px 4px 8px 0px rgba(0,0,0,.1)` 한 단계로 정리한다.

Radius
극단만 남긴 곡률
토큰은 `--f-radius-full` 하나뿐이다. 이미지와 카드에는 inline `0`만 사용해 직선적인 긴장감을 유지한다.

작은 depth는 `--f-shadow-small`, 이미지 어둡힘은 `--f-color-overlay`와 `--f-color-overlay-darker`가 담당한다.
06 · Patterns

Dark hero, red CTA, white relief sections

레이아웃은 전면 hero, 다크/라이트 섹션 교차, 좌측 또는 하단 오버레이 배치가 핵심이다. 컴포넌트는 pill CTA와 flat media frame만으로 인상을 세운다.

Full-viewport hero
F80
`--f-color-gradient-shadow-light`로 이미지 위 텍스트 대비를 확보하고, headline은 FerrariSans와 white만 사용한다.
Hero는 자동차를 무대 위 오브젝트처럼 다룬다. 다크 오버레이가 올라가도 레드를 전면에 깔지 않는다.
CTA system
Primary는 red gradient, secondary는 border-only, focus는 `#F6E500` outline. 세 가지가 Ferrari 버튼 시스템의 대부분을 차지한다.
Light relief section
White surface, black type
라이트 섹션은 `--f-color-background-0`와 `--f-color-background-10` 안에서만 등장한다.
기본 모드는 dark다. 라이트 섹션은 콘텐츠 호흡을 주기 위한 relief로만 사용하고, 토글 UI는 없다.
Layout rules
  • Full-viewport hero (100vw × 100vh)
  • 다크/라이트 섹션 전환
  • 텍스트는 좌측 혹은 하단 오버레이
  • 카드와 이미지 컨테이너는 flat edge
Ferrari는 토큰 수보다 배치의 절제가 더 중요하다. 그리드보다 여백 리듬과 배경 전환이 인상을 만든다.

08 · Rules

      
Do / Don’t

      

        Ferrari의 가장 흔한 실수는 레드를 욕심내는 것이다. 토큰보다도 사용 비율과 배경 선택이 브랜드를 결정한다.
      


      

        

          

            
Do

            
긴장감을 만드는 쪽으로 사용

            
    
              
  • 배경은 `#181818` 또는 white 계열로 유지하고 레드는 포인트에만 올린다.
    • 
              
  • CTA는 `linear-gradient(180deg, #a00c01, #da291c 64%)`와 pill radius를 함께 쓴다.
    • 
              
  • 타이포그래피는 FerrariSans 단일 시스템과 28px → 9px 스케일을 유지한다.
    • 
              
  • 라이트 섹션은 relief 용도로만 제한적으로 사용한다.
    • 
            

          


          

            
Don’t

            
이탈리아 레스토랑처럼 만들지 않기

            
    
              
  • `#DA291C`를 섹션 배경 전체에 깔지 않는다.
    • 
              
  • 4px, 8px 같은 중간 radius를 카드에 추가해 분위기를 흐리지 않는다.
    • 
              
  • 밝은 회색 ramp를 무시하고 임의의 레드/블랙을 섞어 쓰지 않는다.
    • 
              
  • 포커스 색상을 삭제해 다크 배경 접근성을 떨어뜨리지 않는다.
    •
