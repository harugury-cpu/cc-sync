02. Typography

기본 UI는 Cera Pro, 브랜드 모드는 Typewell

현재 global CSS는 `Cera Pro`를 일반 UI 폰트로 쓰고, 브랜드 모드에서만 `LEGO Typewell`을 토큰에 올린다. display/heading scale도 대부분 `700` weight 중심이다.

Cera Pro

기본 UI font. `100 / 400 / 500 / 700 / 900`와 italic 변형이 self-hosted `woff2`로 로드된다.

LEGO Typewell

`[data-mode="brand"]`에서 `--ds-font-font-family`를 대체한다. 일반 UI가 아니라 브랜드성 headline 레이어용이다.

Noto Sans KR / JP / SC

`html[lang="ko"]`, `ja`, `zh`에서 locale override로 들어간다. 지역별 폰트 분기가 전역 토큰에 내장되어 있다.

04. Tokens

8pt grid가 아니라 11px부터 시작하는 비정형 rhythm

spacing, radius, shadow, breakpoints까지 전부 `--ds-*`로 정리되어 있지만, 값은 흔한 SaaS 템플릿보다 훨씬 LEGO 특유의 리듬을 가진다.

Spacing Insight

`11 / 18 / 22 / 25 / 29 / 43 / 50 / 58 / 66 / 86 / 100px`처럼 비등간격 scale. 8pt로 재정규화하면 LEGO의 rhythm이 사라진다.

Radius Insight

buttons는 `999px` pill, cards는 `8 / 12 / 16px` 계열. 화면 전체가 둥글기보다 역할별로 반경이 분리된다.

Container

`.Layout_layout__7aSvA`가 `max-width: 100em`을 쓰므로 최대 콘텐츠 폭은 `1600px`다.

Motion Insight

전역 애니메이션보다 컴포넌트별 timing token 사용. `.25s linear`, `.3s`, `.4s`, `.5s`가 자주 보인다.

05. Components

실제 클래스에서 읽은 컴포넌트 레시피

DS utilities와 CSS Modules가 섞여 있으므로, 디자인을 따라가려면 토큰만이 아니라 대표 클래스 패턴도 같이 이해해야 한다.

Primary Pill Button

Add to bag
.sk-button는 background를 --ds-color-action-primary-enabled에서 받는다.
.sk-button--medium는 height `60px`, 좌우 padding `22px`, font-size `1rem`.
즉 default CTA는 로고 레드가 아니라 블루 action token이다.
.sk-button { background-color: var(--ds-color-action-primary-enabled); border-radius: var(--ds-border-radius-round); color: var(--ds-color-text-on-primary); } .sk-button--medium { --_button-height: var(--ds-layout-size-500); --_button-padding: var(--ds-layout-spacing-200); font-size: 1rem; font-weight: 500; }

Display / Heading Utilities

Build your own headline
.display-*, .heading-*는 모두 font-weight: 700 계열이다.
사이즈는 모두 --ds-screen-text-font-size-* 토큰에서 가져온다.
기본 폰트는 locale / mode에 따라 --ds-font-font-family가 바뀐다.
.display-sm, .display-md, .display-lg, .heading-sm, .heading-md, .heading-lg, .heading-xl, .heading-2xl, .heading-3xl, .heading-4xl, .heading-5xl, .heading-6xl { font-family: var(--ds-font-font-family); font-weight: 700; }

Content Card Shell

ContentCard
A flexible editorial card
`gap: var(--ds-spacing-xs)`와 `max-width: 756px`가 현재 card shell의 기본 rhythm이다.
표준 card shell은 세로 flex + `gap: 22px` 구조다.
editorial variant는 padding이 breakpoint에 따라 확장된다.
radius보다 spacing과 surface hierarchy가 시각 구분에 더 중요하다.
.ContentCard_contentCard__RaobR.ContentCard_standard__WGdOH { display: flex; flex-direction: column; gap: var(--ds-spacing-xs); max-width: 756px; }

06. Voice + Rules

카피는 분류를 우선하고, 시각은 구조적 플레이를 만든다

HTML에서 보이는 텍스트도 브랜드 감성보다 탐색 정보에 더 집중한다. tone은 활기차지만 문장은 차분한 commerce taxonomy다.

Navigation Verbs
Shop / Discover / Help
행동 유도는 짧고 직접적이다. 슬로건 대신 탐색 가능한 메뉴 구조를 우선한다.
"Shop" · "Discover" · "Help"
Taxonomy First
theme, age, price, interests
카피의 역할은 감정 자극보다 분류다. 사용자는 카테고리와 필터를 빠르게 읽어야 한다.
"Sets by theme" · "Age" · "Price ranges" · "Interests"
Explicit Qualifiers
나이와 가격을 숨기지 않는다
`1.5+`, `18+`, `Under $25`처럼 선택 판단에 필요한 qualifiers를 노출한다.
"1.5+" · "18+" · "Under $25" · "Over $100"
Trademark Discipline
파트너 브랜드명을 정확히 유지
라이선스 IP가 많아서 브랜드명 표기 규율이 강하다. 카피도 playful하되 naming은 엄격하다.
"LEGO® Animal Crossing™" · "Nike x LEGO® Collection"

Do

  • 로고 레드와 기본 액션 블루를 분리한다.
  • 기본 UI는 `Cera Pro`, 브랜드 display는 `LEGO Typewell`로 나눈다.
  • spacing은 `11 / 18 / 22 / 25 / 29 / 43...` 실제 scale을 유지한다.
  • surface는 `white / subdued / muted` 3층 구조로 정리한다.
  • buttons는 pill, cards는 `8~16px` 반경에서 관리한다.

Don't

  • 모든 CTA를 `#E3000B`로 통일하지 않는다.
  • CSS에 없는 `#FFCF00`, `#006DB7`를 사실처럼 채우지 않는다.
  • spacing을 8pt grid로 재정규화하지 않는다.
  • body까지 `LEGO Typewell`로 덮지 않는다.
  • dark theme를 기본 페이지 분위기로 오해하지 않는다.
