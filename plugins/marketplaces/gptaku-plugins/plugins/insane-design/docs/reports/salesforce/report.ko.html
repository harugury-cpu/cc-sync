Salesforce — 실제 CSS 기반 분석

salesforce.com의 디자인 시스템 분석 — SDS(Salesforce Design System) --sds-g-* 토큰 직접 추출, 브랜드 블루(#0176D3) CTA, 네이비 텍스트(#032D60)의 엔터프라이즈 SaaS 미학.

Quick Start 5분 안에 Salesforce처럼 만들기
1 · 폰트 body { font-family: "Salesforce Sans", "Helvetica Neue", Arial, sans-serif; font-weight: 400; }
2 · 배경 + 텍스트 :root { --bg: #FFFFFF; --fg: #032D60; } body { background: var(--bg); color: var(--fg); }
3 · 브랜드 블루 :root { /* SDS 토큰 직접 추출 */ --sds-g-color-brand- inverse-contrast-1: #0176D3; --brand: #0176D3; }
절대 하지 말아야 할 것 하나: 밝은 파랑 #1B96FF를 버튼 배경으로 쓰는 것. Salesforce CTA 버튼은 #0176D3(진한 파랑)이고, #1B96FF는 라이트 모드 텍스트 링크용이다. 대비 불충분으로 WCAG 실패.

13 · Do / Don't

Do / Don't

SDS 네임스페이스를 지키는 것이 핵심 — 임의 축약 금지.

DO

  • CTA는 #0176D3 (진한 파랑) — #1B96FF는 텍스트 링크용
  • CSS 변수명 --sds-g-* 원본 그대로 — 축약하거나 일반화 금지
  • 텍스트는 네이비 #032D60 — 순검정이 아닌 브랜드 네이비
  • 블루 tint #EEF4FF를 섹션 배경으로 — 흰 섹션과 블루 섹션 구분
  • 아이콘은 #0176D3 파랑 — feature 카드에서 브랜드 색 일관 사용

DON'T

  • #1B96FF를 버튼 background로 쓰지 않는다 — 라이트 모드 대비 불충분
  • --sds-g-* prefix를 제거하지 않는다 — SDS 네임스페이스 필수
  • 진한 검정(#000) 배경을 기본으로 쓰지 않는다 — Salesforce는 라이트 퍼스트
  • 8단계 블루 스케일을 무시하고 임의 파랑 만들지 않는다 — 체계가 있음
  • font-weight 750 이상을 일반 텍스트에 쓰지 않는다 — 가독성 저하
