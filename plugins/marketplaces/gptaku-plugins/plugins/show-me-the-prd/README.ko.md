# show-me-the-prd

> **바이브코더를 위한 인터뷰 기반 PRD 생성기 — 한 문장 아이디어에서 4종 디자인 문서까지.**

아이디어 한 줄을 완성된 기획 문서로 만들어줍니다. 5–6개의 구조화된 질문에 답하면, AI 코딩 도구에 바로 넘길 수 있는 4종 문서가 완성됩니다.

## 빠른 시작

### 1. 마켓플레이스 등록

```
/plugin marketplace add https://github.com/fivetaku/gptaku_plugins.git
```

### 2. 플러그인 설치

```
/plugin install show-me-the-prd
```

### 3. Claude Code 재시작

플러그인은 세션 시작 시 로드됩니다. 설치 후 한 번 재시작하세요.

### 4. 기획 시작

```
/show-me-the-prd 사진 정리 앱을 만들고 싶어
```

또는 자연어로도 됩니다:

```
할일 관리 앱 PRD 만들어줘
```

## 왜 show-me-the-prd인가

- **기획 경험 불필요** — 모든 질문에 전문 용어 없이 설명과 장단점이 포함되어 있어요
- **AI가 리드, 내가 결정** — AI가 실시간으로 선택지를 조사해서 제안해요. 직접 검색할 필요 없어요
- **리서치 기반, 추측 아님** — 기능 목록, 스택 비교, 복잡도 안내 모두 실시간 웹 검색 결과를 반영해요
- **4종 문서, 하나가 아니라** — PRD, 데이터 모델, Phase 계획, AI 프로젝트 스펙이 하나의 일관된 세트로 생성돼요
- **진짜 제품 지향** — 각 Phase는 로컬 목업이 아닌 실제 배포 + 실제 인증 + 실제 DB를 목표로 해요
- **기존 기획도 OK** — 이미 있는 기획서나 메모를 공유하면, 부족한 부분만 찾아서 채워줘요

## 작동 방식

```
한 문장 아이디어
       |
       v
  [ 인터뷰 ]  ←── 질문 사이사이에 실시간 웹 리서치 실행
       |
   Turn 1: 아이디어 구체화 (1–3 질문)
   Turn 2: 핵심 기능 + MVP 범위 선택
   Turn 3: 데이터 모델 확인
   Turn 4: Phase 분리 확인
   Turn 5: 기술 스택 + 인증 방식 선택
       |
       v
  [ 4종 문서 ]
       |
       +── PRD/01_PRD.md            뭘 만드는지, 누가 쓰는지
       +── PRD/02_DATA_MODEL.md     핵심 데이터 구조 (개념적 ERD)
       +── PRD/03_PHASES.md         단계별 개발 계획 + 시작 프롬프트
       +── PRD/04_PROJECT_SPEC.md   AI 규칙 + "절대 하지 마" 목록
       +── PRD/README.md            네비게이션 가이드
```

인터뷰의 모든 선택지에는 일상 언어로 된 설명, 장점, 단점, 복잡도 안내(간단해요 / 좀 복잡해요 / 많이 복잡해요)가 포함됩니다. 고르기 어려우면 **(추천)** 표시된 걸 선택하면 됩니다.

## 주요 기능

### 인터뷰 기반 기획

개발 용어를 몰라도 괜찮아요. "인증 방식을 선택하세요" 대신 "로그인은 어떻게 할까요?"라고 물어보고, "소셜 로그인 (구글/카카오)", "이메일+비밀번호" 같은 구체적인 선택지를 보여줍니다.

### 실시간 웹 리서치

각 인터뷰 턴 사이에 관련 기능, 흔한 실수, 현재 베스트 프랙티스 스택을 라이브로 검색합니다. 선택지에 보이는 내용은 실제 검색 결과를 반영한 것이에요.

### 복잡도 안내

기능 선택 단계에서 각 기능의 복잡도를 안내해요:

| 표시 | 의미 |
|------|------|
| 간단해요 | 대부분의 프레임워크에서 바로 구현 가능 |
| 좀 복잡해요 | 외부 서비스 연동 필요, 비용 발생 가능 |
| 많이 복잡해요 | 첫 버전에서는 빼는 것을 추천 |

### 4종 산출물 문서

| 문서 | 내용 | 활용 시점 |
|------|------|----------|
| `01_PRD.md` | 제품 목표, 사용자 스토리, 기능 목록 | 프로젝트 시작 시 공유 |
| `02_DATA_MODEL.md` | 핵심 엔티티와 관계 | DB 설계 시 공유 |
| `03_PHASES.md` | 단계별 계획 + 시작 프롬프트 | 개발 순서 참고 |
| `04_PROJECT_SPEC.md` | AI 행동 규칙 + "절대 하지 마" 목록 | **매 세션마다** AI에게 공유 |

### 기획 보강 모드

이미 기획서나 메모가 있다면, 실행 전에 먼저 공유해주세요. 플러그인이 4종 문서 기준으로 빠진 부분을 찾아내고, 필요한 질문만 골라서 진행합니다.

## 명령어

| 명령어 | 설명 |
|--------|------|
| `/show-me-the-prd [아이디어]` | 아이디어를 포함해서 인터뷰 시작 |

- "기획서 만들어줘"
- "쇼미더피알디"
- "앱 기획해줘"
- "[아이디어] 기획해줘"
