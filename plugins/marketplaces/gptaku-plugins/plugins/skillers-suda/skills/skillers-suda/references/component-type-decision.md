# 컴포넌트 타입 판단 기준

## 개요

인터뷰 결과를 바탕으로 스킬(Skill) / 에이전트(Agent) / 커맨드(Command) 중 적합한 형태를 자동 판단한다.

## 비교 테이블

| 기준 | 스킬 (Skill) | 에이전트 (Agent) | 커맨드 (Command) |
|------|-------------|-----------------|-----------------|
| 트리거 | 자동 감지 (description 매칭) | 자동 감지 (description 매칭) | 유저가 /명령어로 직접 실행 |
| 컨텍스트 | 기존 대화에 녹아듦 | 독립 컨텍스트 (별도 프로세스) | 기존 대화 내에서 실행 |
| 실행 방식 | 메인 Claude가 직접 실행 | 별도 Claude 에이전트가 자율 실행 | 메인 Claude가 직접 실행 |
| 복잡도 | 단순~중간 | 중간~복잡 | 단순~중간 |
| 도구 접근 | 모든 도구 사용 가능 | 허용된 도구만 | 허용된 도구만 |
| 상태 | 대화 상태 공유 | 자체 상태 관리 | 대화 상태 공유 |
| 적합한 작업 | 한 가지 일을 처리 | 여러 단계를 자율적으로 수행 | 인자 기반으로 분기 처리 |

## 자동 판단 로직

### 스킬을 선택하는 경우
```
IF 아래 조건 중 하나라도 해당:
  - 한 가지 목적의 워크플로우
  - 대화 맥락 안에서 처리 가능
  - 사용자와 대화하면서 진행해야 함
  - AskUserQuestion으로 중간 확인 필요
THEN → 스킬
```

예시:
- "번역 스킬" → 대화 내에서 번역 워크플로우 실행
- "코드 리뷰 스킬" → 파일 읽고 분석 후 리포트
- "문서 생성 스킬" → 인터뷰 후 문서 작성

### 에이전트를 선택하는 경우
```
IF 아래 조건 중 하나라도 해당:
  - 자율적으로 여러 단계를 실행해야 함
  - 독립적인 컨텍스트가 필요함
  - 오래 걸리는 작업 (백그라운드 실행)
  - 다른 에이전트/스킬과 병렬 실행
  - 제한된 도구만 접근해야 안전함
THEN → 에이전트
```

예시:
- "보안 감사 에이전트" → 전체 코드베이스 스캔
- "리서치 에이전트" → 웹 검색 + 분석 + 요약
- "QA 테스트 에이전트" → 자율적 테스트 실행

### 커맨드를 선택하는 경우
```
IF 아래 조건 중 하나라도 해당:
  - 유저가 명시적으로 호출해야 함 (/command)
  - 인자에 따라 다른 동작을 함
  - 메뉴/옵션 UI가 필요함
  - 여러 하위 기능을 묶음
THEN → 커맨드
```

예시:
- "/deploy" → 배포 파이프라인 실행
- "/translate en ko file.md" → 인자 기반 번역
- "/project-setup" → 프로젝트 초기 설정 메뉴

## 복합 구성

하나의 플러그인에 여러 컴포넌트를 조합할 수 있다:

```
플러그인/
├── skills/
│   └── main-skill/SKILL.md       # 자동 감지 워크플로우
├── commands/
│   └── command.md                 # /명시적 호출
└── .claude/agents/
    └── agent.md                   # 독립 실행
```

예시: 번역 플러그인
- Skill: "번역해줘" → 자동 감지, 인터뷰 후 번역
- Command: "/translate en ko file.md" → 바로 번역
- Agent: 대규모 번역 → 백그라운드 자율 실행

## 판단 순서도

```
사용자 요청 분석
    │
    ├── 자동으로 감지되어야 하는가?
    │   ├── Yes → 스킬 또는 에이전트
    │   │   ├── 자율 실행이 필요한가?
    │   │   │   ├── Yes → 에이전트
    │   │   │   └── No → 스킬
    │   └── No → 커맨드
    │
    └── 복합 구성이 필요한가?
        ├── Yes → 여러 컴포넌트 조합
        └── No → 단일 컴포넌트
```

## 각 컴포넌트의 파일 구조

### Skill 파일 구조
```yaml
---
name: skill-name
description: This skill should be used when...
---
# 워크플로우 내용
```

### Agent 파일 구조
```yaml
---
name: agent-name
description: Use this agent when...
model: inherit
color: blue
tools: [Read, Write, ...]
---
# 에이전트 시스템 프롬프트
```

> 에이전트 파일 표준 포맷 상세: `docs/agent-file-standard.md` 참조.
> 확장 메타데이터 (`source`, `created`, `recommended-for`)를 frontmatter에 포함하면 kkirikkiri와의 연동이 자동으로 동작합니다.

### Command 파일 구조
```yaml
---
name: command-name
description: 한 줄 설명
allowed-tools: [Read, Write, ...]
---
# 커맨드 실행 로직
```
