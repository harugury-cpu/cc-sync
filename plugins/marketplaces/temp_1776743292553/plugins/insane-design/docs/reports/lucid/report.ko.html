Lucid — 실제 CSS 기반 분석

lucid.app의 디자인 시스템 분석 — 골드(#D7BE96) 브랜드, 다크 #0F0F0F 배경, GTAmerica 300 light body의 세련된 B2B 협업 툴 미학.

Quick Start 5분 안에 Lucid처럼 만들기
1 · 폰트 + weight body { font-family: "GTAmerica", "IBM Plex Sans", -apple-system, Helvetica, sans-serif; font-weight: 300; /* light */ }
2 · 다크 배경 :root { --bg: #0F0F0F; --fg: #FFFFFF; --bg-card: #1C1C1C; } body { background: var(--bg); color: var(--fg); }
3 · 브랜드 골드 :root { --color-primary-colors-gold: #D7BE96; --border-brand-gold: #D7BE96; --text-brand-gold: #D7BE96; }
절대 하지 말아야 할 것 하나: 골드(#D7BE96)를 라이트 배경에 쓰는 것. Lucid 골드는 다크 배경(#0F0F0F)과의 대비에서만 효과가 살아난다. 라이트 배경 위 골드는 WCAG 대비율이 1.9:1 수준으로, 가독성이 전혀 없다.

03 · Typography

타이포그래피

GTAmerica 300 light — 세련된 B2B 마케팅 아이덴티티의 핵심.

Font stack
마케팅GTAmerica — Grilli Type · 유료 · weight 300 (light body)
DisplaySchnyderS, SchnyderM — 대형 헤드라인 (serif, 골드와 조합)
앱 내부Lucid Sans VF — Lucid 전용 variable font · weight 300
Weight 정의light = 300 (본문 핵심) · normal = 400 · semibold = 600
대체재마케팅 → Inter · Schnyder → Playfair Display
핵심 원칙: Lucid 마케팅 본문은 반드시 font-weight: 300이다. 400으로 올리는 순간 Lucid의 세련된 경량감이 사라진다. light weight가 이 브랜드의 정체성이다.

13 · Do / Don't

Do / Don't

Lucid 미학의 핵심 원칙과 치명적 실수.

✓ DO

  • 배경은 #0F0F0F — 순검정이 아닌 약간 따뜻한 near-black
  • 골드(#D7BE96)는 반드시 다크 배경 위에서 — 라이트 배경 위 가독성 불가
  • 본문 weight는 300 (light) — GTAmerica light가 Lucid 세련됨의 핵심
  • CSS 토큰명 --color-primary-colors-gold 원본 유지 — 임의 축약 금지
  • Schnyder serif를 display에 — 골드와 세리프의 조합이 고급스러움
  • feature-card hover에 골드 border + gold-glow 섀도우 함께 적용

✗ DON'T

  • 골드를 라이트 배경에 쓰지 않는다 — 대비 1.9:1로 WCAG 불통과, 가독성 없음
  • body weight를 400으로 올리지 않는다 — 300이 Lucid 마케팅 정체성
  • 앱 컬러(#00AD56 green, #DC8100 orange)를 마케팅 사이트에 그대로 쓰지 않는다
  • 골드와 흰색만으로 팔레트 구성하지 않는다 — 다크 그레이 계층 (#1C1C1C, #323232)이 필수
  • Lucid Sans VF(앱 폰트)를 마케팅에 적용하지 않는다 — GTAmerica가 마케팅용
  • 토큰명 임의 단축 금지 — --gold가 아닌 --color-primary-colors-gold
