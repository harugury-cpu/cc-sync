Design System Report

Microsoft — Fluent Design 2

https://microsoft.com 이 실제로 서빙하는 CSS에서 직접 뽑은 디자인 토큰·타이포그래피·컴포넌트 스펙. 마케팅 사이트는 레거시 MWF 구조를 사용하지만, 디자인 시스템은 Fluent Design System 2 원칙을 따른다.

Quick Start 5분 안에 Microsoft처럼 만들기 — 3가지만 하면 80%
1 · 폰트 + weight body { font-family: "Segoe UI Variable", "Segoe UI", system-ui, -apple-system, sans-serif; font-weight: 400; }
2 · 배경 + 텍스트 :root { --bg: #FFFFFF; --fg: #242424; /* 순흑 아님 */ } body { background: var(--bg); color: var(--fg); }
3 · 브랜드 블루 :root { --brand: #0078D4; /* Fluent 2 brand-60 CTA · 링크 · focus */ }
절대 하지 말아야 할 것 하나: 브랜드 블루로 #1BA1E2(Windows 8 시대 구버전)를 쓰는 것. Microsoft는 2019년 이후 Fluent Design으로 전환하며 #0078D4를 canonical CTA blue로 고정했다. 구버전 파란색을 쓰면 즉시 구식처럼 보인다.

03 · Typography

타이포그래피

Segoe UI Variable(Windows 11)과 Segoe UI(이전 버전)의 차이를 이해하는 것이 중요하다. Fluent 2 타이포 스케일에서 heading은 weight 600이 기본이다.

Font stack
Display font Segoe UI Variable — Windows 11+ 기본 탑재, 가변 폰트
Body font Segoe UI — Windows 기본 탑재 (고정 폰트)
Code font Cascadia Code, Consolas, monospace
Weight normal / bold 400 / 600
플랫폼별 fallback: Segoe UI Variable은 Windows 11에 탑재됨. Windows 10 이하는 Segoe UI로, macOS/Linux는 system-ui로 fallback한다. 비Windows 환경에서도 시스템 UI 폰트가 자연스럽게 적용된다.

CSS · Drop-in

루트 스타일시트에 바로 붙여넣으면 Fluent 2 디자인 토큰을 그대로 쓸 수 있다.

:root {
  /* Fonts */
  --font-family: "Segoe UI Variable", "Segoe UI", system-ui, -apple-system, sans-serif;
  --font-family-code: "Cascadia Code", Consolas, monospace;
  --font-weight-normal: 400;
  --font-weight-bold: 600;

  /* Brand (Fluent 2 blue ramp) */
  --brand-10: #EFF6FC;
  --brand-30: #C7E0F4;
  --brand-50: #2B88D8;
  --brand-60: #0078D4;  /* ← canonical */
  --brand-70: #106EBE;  /* hover */
  --brand-80: #005A9E;  /* active */
  --brand-90: #004578;

  /* Surfaces */
  --bg-page: #FFFFFF;
  --bg-secondary: #F9F9F9;
  --text: #242424;  /* 순흑 아님 */
  --text-muted: #707070;

  /* Key spacing */
  --space-sm: 12px;
  --space-md: 16px;
  --space-lg: 24px;

  /* Radius */
  --radius-sm: 2px;
  --radius-md: 4px;  /* 버튼 기본 */
  --radius-lg: 8px;

09 · Do / Don't

Do / Don't

Fluent Design System 2를 구현할 때 가장 자주 틀리는 항목들.

DO

  • 브랜드 블루는 #0078D4(brand-60)을 써라.
  • 텍스트는 #242424를 써라 — 순흑이 아닌 Fluent의 dark gray다.
  • 버튼 weight는 600을 써라 — Fluent 2 기준이다.
  • 버튼 border-radius: 4px을 기본으로 써라 — Fluent의 기본 radius다.
  • 4px 배수 그리드를 지켜라.

DON'T

  • #1BA1E2(Windows 8 구버전 파랑)를 쓰지 마라 — Fluent 2는 #0078D4다.
  • Arial을 기본 폰트로 두지 마라 — Windows에서는 Segoe UI가 로드된다.
  • 버튼 radius를 0으로 두지 마라 — Fluent는 4px이 기본이다.
  • 무거운 font-weight: 700을 heading에 쓰지 마라 — Fluent heading은 600이다.
  • 배경을 진한 회색으로 만들지 마라 — 기반은 #FFFFFF다.
Common Mistakes
틀린 값 올바른 값 이유
#1BA1E2 #0078D4 Windows 8 시대 구버전 파랑. Fluent 2로 전환하며 #0078D4로 고정됐다.
#000000 #242424 Fluent 텍스트는 따뜻한 dark gray. 순흑은 Fluent 팔레트에 없다.
font-weight: 700 font-weight: 600 Fluent 2 heading의 최대 weight는 600. 700은 과도하게 두껍다.
border-radius: 0 border-radius: 4px Fluent 버튼은 4px radius. 완전 사각형은 Fluent 원칙이 아니다.
font-family: Arial Segoe UI Variable, Segoe UI Arial은 Windows에서 Segoe UI로 대체된다. fallback 순서를 맞춰라.
