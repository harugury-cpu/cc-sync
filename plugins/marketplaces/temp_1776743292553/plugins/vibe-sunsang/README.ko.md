# vibe-sunsang (바선생)

<p align="center">
  <img src="assets/logo.jpeg" alt="vibe-sunsang" width="240">

> **Claude Code와 나눈 모든 대화를 성장 엔진으로 바꾸는 AI 멘토 에이전트.**

바선생은 당신이 Claude Code와 나눈 대화를 자동으로 수집하고, 읽기 좋은 형태로 변환하고, 분석합니다. 그래서 더 나은 AI 협업자로 성장할 수 있게 도와줍니다.

## 빠른 시작

### 1. 설치

```
/plugin marketplace add https://github.com/fivetaku/gptaku_plugins.git
/plugin install vibe-sunsang
```

설치 후 **Claude Code를 재시작**하세요.

### 2. 온보딩 실행

아무 프로젝트에서:

```
/vibe-sunsang 시작
```

온보딩이 하는 일:

1. `~/vibe-sunsang/` 워크스페이스 자동 생성 (`CLAUDE.md`, `.gitignore`, `git init` 포함)
2. `~/.claude/projects/`에서 Claude Code 대화 기록 탐색
3. 발견된 프로젝트에 읽기 좋은 이름 지정
4. 워크스페이스 유형 분류 (Builder / Explorer / Designer / Operator)
5. 모든 JSONL 대화를 Markdown으로 첫 변환
6. 바로 멘토링 또는 성장 리포트를 시작할지 선택

### 3. 매주 사용하기

```
/vibe-sunsang 변환      ← 이번 주 대화 변환
/vibe-sunsang 멘토링    ← AI 활용 패턴 코칭
/vibe-sunsang 성장      ← 성장 리포트 생성
/vibe-sunsang 지식      ← 레벨 시스템과 개념 학습
```

## 왜 바선생인가요?

- **대화 기록이 이미 있습니다** — Claude Code는 모든 세션을 JSONL로 저장합니다. 바선생은 그 파일을 읽기 때문에 새로 기록할 것이 없습니다
- **"무엇을 만들었나"가 아닌 "어떻게 만들었나"** — 요청 품질, 오류 대응 습관, 전략적 사고를 분석합니다. 결과물 양이 아니라요
- **워크스페이스 유형에 맞는 분석** — 코딩, 리서치, 기획, 자동화 중 무엇을 하는지에 따라 레벨 시스템이 달라집니다
- **하나의 점수가 아닌 6가지 차원** — DECOMP, VERIFY, ORCH, FAIL, CTX, META를 독립적으로 측정해서 어떤 역량을 키워야 할지 정확히 알 수 있습니다
- **종단 추적** — 성장 리포트마다 `TIMELINE.md`가 업데이트되어 단일 세션이 아닌 수개월 단위의 성장을 볼 수 있습니다
- **비개발자 친화적** — 비유와 일상 예시로 설명합니다. 전문 용어 장벽이 없습니다

## 작동 방식

```
Claude Code 대화 로그 (~/.claude/projects/)
                    |
                    | /vibe-sunsang 변환
                    v
     읽기 좋은 Markdown 파일 (~/vibe-sunsang/conversations/)
                    |
                    | 6축 분석
                    | (DECOMP / VERIFY / ORCH / FAIL / CTX / META)
                    v
     멘토링 세션  ←→  성장 리포트  ←→  개념 학습
                    |
                    | Fit Score  ×  워크스페이스 유형별 가중치
                    v
     레벨 판정: L1.0 – L7.0 (0.5 단위 세분화)
                    |
                    v
     TIMELINE.md (종단 성장 추적)
```

대화 데이터와 플러그인 코드는 완전히 분리된 디렉토리에 저장됩니다. 플러그인을 업데이트해도 데이터가 영향받지 않습니다.

## 기능 모드

### 시작 (`시작`)

`~/vibe-sunsang/` 워크스페이스를 생성하고, 프로젝트 이름을 매핑하고, 워크스페이스 유형을 분류하고, 첫 번째 전체 변환을 실행합니다.

---

### 변환 (`변환`)

Claude Code의 JSONL 로그를 프로젝트별로 정리된 읽기 좋은 Markdown 파일로 변환합니다.

| 명령 | 동작 |
|------|------|
| `/vibe-sunsang 변환` | 증분 변환 — 새 세션만 변환 |
| `/vibe-sunsang 변환 force` | 전체 재변환 |
| `/vibe-sunsang 변환 <프로젝트명>` | 특정 프로젝트만 변환 |

변환된 각 파일에는 메타데이터가 포함됩니다: 날짜, 토큰 수, 사용 모델, 호출된 도구 목록. `INDEX.md`에서 전체 현황을 한눈에 파악할 수 있습니다.

---

### 멘토링 (`멘토링`)

서로 다른 기술 차원에 집중하는 4가지 코칭 모드:

| 모드 | 트리거 | 하는 일 | 중심 축 |
|------|--------|---------|---------|
| **A — 요청 품질 코칭** | "요청 코칭해줘" | 내 요청을 A~D 등급으로 채점하고 개선 예시 제시 | DECOMP + CTX |
| **B — 안티패턴 진단** | "뭘 잘못하고 있는지" | 구체적 사례와 해결 전략이 담긴 나쁜 습관 탐지 | FAIL + VERIFY |
| **C — 개념 학습** | "이게 뭐야" | 최근 세션에 나온 개념을 비유와 예시로 쉽게 설명 | META |
| **D — 종합 코칭** (기본) | "멘토링해줘" | 6축 전체 점검 + 레벨 판정 + 행동 계획 | 6축 전체 |

모드 D는 텍스트 레이더 차트와 3단계 행동 계획을 제공합니다: 지금 당장 할 것 1가지, 이번 주에 할 것 1가지, 한 달 안에 목표할 것 1가지.

코칭 결과는 `~/vibe-sunsang/exports/mentor-YYYY-MM-DD.md`에 저장할 수 있습니다.

---

### 성장 리포트 (`성장`)

전용 `growth-analyst` 서브에이전트에게 분석을 위임하여 메인 컨텍스트를 보호합니다. 에이전트는 세션 파일을 읽고, 6개 차원 전체를 채점하고, 리포트를 작성합니다.

**분석 범위 옵션:**

| 범위 | 기간 |
|------|------|
| 이번 주 | 최근 7일 |
| 최근 2주 (추천) | 최근 14일 |
| 이번 달 | 최근 30일 |
| 특정 프로젝트 | 프로젝트 전체 기간 |

**리포트에 포함되는 내용:**

1. 기본 통계 — 세션 수, 메시지 수, 토큰 사용량
2. 6축 기술 차원 분석 (점수, 가중치, 핵심 근거)
3. 텍스트 레이더 차트
4. 게이트 조건 충족 상태 테이블 (다음 레벨 진입 요건)
5. 안티패턴 탐지 (워크스페이스 유형에 맞는 기준 적용)
6. 이전 리포트 전체와의 종단 비교
7. 진행 바 포함 레벨 카드
8. 레벨 상승 시 승급 메시지 (L4→L5는 "80%의 벽 돌파" 특별 이벤트)
9. `TIMELINE.md` 자동 업데이트

---

### 지식 (`지식`)

바선생 프레임워크 자체에 대한 질문에 답변합니다:

| 주제 | 설명 |
|------|------|
| 레벨 시스템 | 7단계 성장 레벨(0.5 단위)과 각 단계의 의미 |
| 6축 기술 차원 | DECOMP, VERIFY, ORCH, FAIL, CTX, META 정의와 예시 |
| 워크스페이스 유형 | Builder / Explorer / Designer / Operator 유형 설명 |
| 안티패턴 | 워크스페이스 유형별로 주의해야 할 나쁜 습관 |
| 요청 품질 | 더 나은 요청을 하는 방법 |
| 성장 지표 | Fit Score 공식, 게이트 조건, 유형별 가중치 |

## 레벨 시스템

### 6대 기술 차원

| 코드 | 기술 차원 | 한 줄 정의 |
|------|----------|-----------|
| **DECOMP** | 작업 분해 | 복잡한 요청을 AI가 처리 가능한 단위로 나누는 능력 |
| **VERIFY** | 검증 전략 | AI 출력물을 비판적으로 검토하고 품질을 확인하는 능력 |
| **ORCH** | 오케스트레이션 | 도구, 에이전트, 워크플로우를 조합하여 활용하는 능력 |
| **FAIL** | 실패 대응 | 오류, 한계, 예상치 못한 결과에 대처하는 능력 |
| **CTX** | 맥락 관리 | AI에게 적절한 배경 정보, 제약 조건, 목표를 제공하는 능력 |
| **META** | 메타인지 | 자신의 AI 활용 패턴을 인식하고 전략적으로 조정하는 능력 |

### 워크스페이스 유형별 가중치

실제로 무엇을 하는지에 따라 분석 가중치가 달라집니다:

| 기술 차원 | Builder | Explorer | Designer | Operator |
|----------|---------|----------|----------|----------|
| DECOMP | **25%** | 15% | 20% | 15% |
| VERIFY | **25%** | 15% | 15% | 20% |
| ORCH | 15% | 10% | 10% | **25%** |
| FAIL | 15% | **20%** | 10% | **20%** |
| CTX | 10% | **20%** | **25%** | 10% |
| META | 10% | **20%** | **20%** | 10% |

### 7단계 성장 레벨 (유형별 레벨명)

| 레벨 | Builder | Explorer | Designer | Operator |
|------|---------|----------|----------|----------|
| L1 | Observer (관찰자) | Asker (질문자) | Dreamer (꿈꾸는 사람) | User (사용자) |
| L2 | Tinkerer (만지작거리는 사람) | Curious (호기심 많은 사람) | Sketcher (스케치하는 사람) | Repeater (반복자) |
| L3 | Collaborator (협력자) | Digger (파헤치는 사람) | Shaper (다듬는 사람) | Optimizer (최적화자) |
| L4 | Pilot (조종사) | Investigator (탐구자) | Planner (설계자) | Builder (구축자) |
| L5 | Architect (설계자) | Analyst (분석가) | Strategist (전략가) | Engineer (엔지니어) |
| L6 | Conductor (지휘자) | Synthesizer (통합자) | Director (감독) | Orchestrator (오케스트레이터) |
| L7 | Forgemaster (대장장이) | Scholar (학자) | Visionary (비전가) | Automator (자동화 마스터) |

레벨은 내부적으로 소수점 2자리까지 추적하고 (예: 3.72), 표시 시 0.5 단위로 반올림합니다 (예: 3.5). 판정 기준은 "할 수 있는가"가 아닌 **"일관되게 하는가"**입니다.

### 게이트 조건

| 게이트 | 필수 요건 |
|--------|---------|
| L3 진입 | 맥락 구체성 비율 > 0.5 |
| L4 진입 | 검증 요청 비율 > 0.15 AND 수정 요청 비율 > 0.05 |
| L5 진입 | 도구 다양성 > 8 (또는 오케스트레이션 도구 사용) AND 전략적 논의 비율 > 0.05 |
| L6 진입 | 멀티에이전트 경험 증거 |
| L7 진입 | L6 게이트 통과 + 외부 기여 증거 |

## 추천 루틴

**매주 금요일 (15분)**

```
1. /vibe-sunsang 변환      ← 이번 주 대화 변환
2. 멘토링해줘               ← 이번 주 리뷰
3. 다음 주에 시도할 것 1가지 실천
```

**매월 1회 (30분)**

```
1. /vibe-sunsang 변환 force  ← 전체 재변환
2. 성장 리포트 만들어줘        ← 월간 성장 리포트
3. 이전 리포트와 비교 — 트렌드가 자동으로 분석됩니다
```

## 파일 구조

모든 사용자 데이터는 `~/vibe-sunsang/`에 저장됩니다. 플러그인 코드와 완전히 분리됩니다.

```
~/vibe-sunsang/
├── CLAUDE.md                 ← 워크스페이스 설명 (온보딩이 자동 생성)
├── .gitignore
├── config/                   ← 프로젝트명 매핑, 워크스페이스 유형 설정
├── conversations/            ← 변환된 Markdown 세션
│   └── INDEX.md              ← 전체 프로젝트 인덱스
├── exports/                  ← 성장 리포트, 멘토링 결과 저장
└── growth-log/
    ├── TIMELINE.md           ← 전체 리포트 기반 레벨/6축 변화 타임라인
    └── weekly/               ← 주간 회고 노트
```
