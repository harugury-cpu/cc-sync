# 점수 기준 (정수 1-5, 직관 금지)

각 항목은 1~5점으로 평가하되, 아래 통과 조건에 매핑한다. 직관 점수는 금지한다. scoreReasons에는 어느 조건에 해당해서 그 점수인지 1문장으로 명시한다.

## 소비자 체감도 (consumerImpact)

- 5: 구매 후 매일 체감, 결정적 효익
- 4: 자주 체감, 만족도 큼
- 3: 가끔 체감, 보조적 효익
- 2: 드물게 체감, 의식하지 않으면 모름
- 1: 사양 표기에만 존재, 실제 체감 없음

판단 근거:

- 사용 중 바로 체감되는가
- 자주 겪는 불편과 연결되는가
- 설명 없이도 효익이 이해되는가

## 메시지 차별성 (messageDifferentiation)

- 5: 우리만의 기능 단어 2개 이상 + 경쟁사 자료에서 동일 메시지 0건
- 4: 우리만의 기능 단어 1개 + 경쟁사 0건
- 3: 메시지는 같지만 표현 각도 차이 가능
- 2: 동일 메시지가 경쟁사 1-2건
- 1: 동일 메시지 경쟁사 3건 이상 반복

판단 근거:

- 경쟁사 자료에 같은 메시지가 반복되는가
- 우리 기능/구조/스펙이 다른가
- 기존 방식 대비 명확한 개선이 있는가

## 표현 차별성 (expressionDifferentiation)

- 5: 경쟁사가 안 쓰는 구도/시점/장면 + 정보 구조 재설계 여지
- 4: 경쟁사 표현이 단조롭고 우리는 새 구도 가능
- 3: 일부 다르게 표현 가능
- 2: 표현 차별 여지 적음
- 1: 경쟁사 모두 동일 표현, 차별 어려움

판단 근거:

- 경쟁사 표현이 단순하거나 반복적인가
- 우리 기능을 더 잘 증명하는 장면이 있는가
- 비교, 클로즈업, 도식, 실사용 맥락으로 차별화 가능한가

## 시각화 용이성 (visualEase)

- 5: 한 컷에 핵심 메시지 즉시 전달 가능
- 4: 한 컷 + 작은 라벨로 전달 가능
- 3: 1-2컷이면 전달 가능
- 2: 3컷 이상 또는 영상 필요
- 1: 시각으로 표현 불가, 텍스트 설명 필수

판단 근거:

- 실제 사용 장면으로 보이는가
- 전후/비교/결과물로 표현 가능한가
- 치수/구조/단면/아이콘으로 정리 가능한가

## 증명력 (proofPower)

- 5: 작동 순간 또는 결과물을 사진으로 직접 보여줄 수 있음
- 4: 비교컷 또는 인증 데이터로 증명 가능
- 3: 일러스트/도식으로 증명 가능
- 2: 텍스트 보조 필요
- 1: 증명 어려움, 사용자 경험에만 의존

판단 근거:

- 기능 작동을 직접 보여주는가
- 결과물 비교가 가능한가
- 테스트/인증/수치와 함께 제시 가능한가

## 구매 영향도 (purchaseInfluence)

- 5: 핵심 구매 이유, 결정 직결
- 4: 주요 구매 이유 중 하나
- 3: 구매 이유 중 보조적
- 2: 의사결정에 작은 영향
- 1: 거의 영향 없음

판단 근거:

- 핵심 구매 이유인가
- 구매 전 불안을 줄이는가
- 가격/선택 이유를 정당화하는가

## inputDataConfidence 판정 게이트

분석 시작 시 경쟁사 자료의 양과 깊이를 판정한다.

- high: 경쟁사 자료 3개 이상 + 각 본문 100자 이상 + missingMandatoryElements 1개 이하
- medium: 경쟁사 자료 2개 이상 + 본문 50자 이상
- low: 경쟁사 자료 1개 이하 또는 본문이 거의 비어 있음

경쟁사 자료가 빈약하거나 missingMandatoryElements가 2개 이상이면 high 금지.

## selfReview — 자기 비판과 점수 재조정

점수 1차 산정 후 다음을 점검한다.

- inputDataConfidence가 medium 또는 low이면 messageDifferentiation과 expressionDifferentiation을 1점씩 하향한다.
- 점수 평균이 4.5 이상이면 과대평가 가능성을 자기 비판하고 가장 근거가 약한 항목을 1점 하향한다.
- 위험을 발견하면 코멘트만 남기지 말고 점수와 scoreReasons를 실제로 수정한다.
- selfReview 필드에는 "수정 없음" 또는 "X점 → Y점으로 하향한 이유"를 명시한다.
