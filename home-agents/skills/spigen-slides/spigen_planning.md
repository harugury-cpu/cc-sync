# spigen-slides Planning Guide (v5)

## 역할 분리

| 주체 | 역할 |
|------|------|
| 사용자 | 내용 제공, 구성 승인 |
| AI | 내용 정리, 슬라이드 구성 결정, 생성 |

AI는 내용을 창작하지 않는다. 사용자가 준 내용을 정리할 뿐이다.

---

## Step 1. 내용 수집

### 경로 A: 원본 데이터 있음

문서, 보드 데이터, 채팅 기록, 노트 등 원본이 있으면:
1. 원본을 읽는다
2. 핵심 사실(fact)만 추출한다
3. 추출 결과를 사용자에게 확인받는다

추출 원칙:
- 원본에 없는 내용을 추가하지 않는다
- 수치, 고유명사, 시스템명은 원본 그대로 사용한다
- 요약할 때 의미가 달라지면 원본 문장을 그대로 쓴다

### 경로 B: 사용자 직접 제공

아래 4가지를 묻는다. 이것만 묻는다.

```
Q1. 이 PPT의 목적은 무엇인가요?
    보고 / 교육·안내 / 제안 / 홍보 중 선택 또는 직접 설명

Q2. 청중은 누구인가요?
    (예: 경영진, 팀 내부, 외부 고객)

Q3. 전달할 핵심 내용을 적어주세요.
    형식 무관 — 글머리, 메모, 흐름 설명, 키워드 나열 모두 가능

Q4. 테마: 생략 가능 (기본 = light, 필요 시 dark)
```

내용(Q3)이 충분히 구체적이면 바로 Step 2로 진행한다.
테마를 따로 지정하지 않으면 **light를 기본값**으로 사용한다.

## Step 2. 슬라이드 구성 제안

### 출력 형식

```
=== 슬라이드 구성 ===

[1] 표지 — (제목)
[2] (슬라이드 제목) — (핵심 내용 1줄)
[3] ...

승인하시면 생성을 시작합니다.
```

### 슬라이드 유형 선택

| 상황 | 유형 | 함수 |
|------|------|------|
| 첫 슬라이드 | 표지 | `cover()` |
| 일반 내용 (설명, 나열, 단계) | 헤더+텍스트박스 | `slide()` |
| Before/After 또는 두 항목 비교 | 2단 분리 | `two_col()` |

유형을 먼저 정하지 않는다. 내용을 파악한 뒤 자동 결정한다.

### 슬라이드 수 기준

| 목적 | 권장 장수 |
|------|----------|
| 빠른 보고 | 4~6장 |
| 교육·안내 | 6~10장 |
| 제안서 | 5~8장 |

### Step 3 진입 조건

- [ ] 사용자가 구성을 승인했는가
- [ ] 모든 슬라이드 내용이 사용자 제공 데이터에 근거하는가

하나라도 No → 구성 수정 후 재승인.

---

## 수정 요청 처리

**구성 단계 수정**: 해당 슬라이드만 수정 → 전체 구성 재출력 → 승인  
**생성 후 수정**: 해당 슬라이드만 재생성  
**생성 후 구조 변경**: Step 2부터 재시작
