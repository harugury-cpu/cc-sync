# spigen-slides Planning Guide (v5)

## 역할 분리

| 주체 | 역할 |
|------|------|
| 사용자 | 내용 제공, 구성 승인 |
| AI | 내용 정리, 슬라이드 구성 결정, 생성 |

AI는 내용을 창작하지 않는다. 사용자가 준 내용을 정리할 뿐이다.

---

## Step 1. 내용 수집

### 경로 A: 원본 데이터 있음

문서, 보드 데이터, 채팅 기록, 노트 등 원본이 있으면:
1. 원본을 읽는다
2. 핵심 사실(fact)만 추출한다
3. 추출 결과를 사용자에게 확인받는다

추출 원칙:
- 원본에 없는 내용을 추가하지 않는다
- 수치, 고유명사, 시스템명은 원본 그대로 사용한다
- 요약할 때 의미가 달라지면 원본 문장을 그대로 쓴다

### 경로 B: 사용자 직접 제공

아래 4가지를 묻는다. 이것만 묻는다.

```
Q1. 이 PPT의 목적은 무엇인가요?
    보고 / 교육·안내 / 제안 / 홍보 중 선택 또는 직접 설명

Q2. 청중은 누구인가요?
    (예: 경영진, 팀 내부, 외부 고객)

Q3. 전달할 핵심 내용을 적어주세요.
    형식 무관 — 글머리, 메모, 흐름 설명, 키워드 나열 모두 가능

Q4. 테마: 생략 가능 (기본 = light, 필요 시 dark)
```

내용(Q3)이 충분히 구체적이면 바로 Step 2로 진행한다.
테마를 따로 지정하지 않으면 **light를 기본값**으로 사용한다.

## Step 1.5. 콘텐츠 분석 (내부 — 출력 안 함)

Step 2 구성 전 아래 3단계를 순서대로 수행한다. **컴포넌트는 마지막에 결정한다.**

### ① 사실 추출 — delta 고정값

| 항목 | 처리 |
|-----|------|
| 수치·달성률·퍼센트 | 원본 그대로 고정 |
| 고유명사·시스템명·제품명 | 원본 그대로 고정 |
| 날짜·버전·담당자명 | 원본 그대로 고정 |
| 동사·조사·연결어 | 슬라이드 형식 맞게 최소 조정 허용 |
| 원본에 없는 사실 | 절대 추가 금지 |

### ② 슬라이드 역할 결정 (내용이 먼저, 컴포넌트는 아직 결정하지 않음)

각 슬라이드가 청중에게 **무엇을 하게 만드는가**를 먼저 정한다.

| 역할 | 정의 | 판단 기준 |
|-----|------|---------|
| **설명** | 사실·현황·배경을 전달 | "이게 뭔지" 알려주는 슬라이드 |
| **비교** | 두 상태·대상을 나란히 보여줌 | Before/After, A vs B, 현행 vs 개선 |
| **결정 유도** | 청중이 판단하거나 승인해야 함 | 숫자·조건·결론이 핵심인 슬라이드 |
| **순서** | 단계나 프로세스를 따라가게 함 | 1→2→3 흐름이 있는 슬라이드 |
| **체크/확인** | 완료 여부나 상태를 확인하게 함 | 완료·진행중·대기 항목이 있는 슬라이드 |

역할을 먼저 확정하지 않으면 ③으로 넘어가지 않는다.

### ③ 역할 → 컴포넌트 매핑 (v5 API 기준)

| 역할 | 컴포넌트 | 비고 |
|-----|---------|------|
| 표지 | `cover()` | 항상 첫 슬라이드 |
| 설명 / 순서 / 체크 / 결정 유도 | `slide()` | 헤더 + 본문 텍스트 |
| 비교 | `two_col()` | 좌우 두 컬럼 |

**원칙**: 컴포넌트는 역할의 결과다. 컴포넌트가 역할을 강요하면 안 된다.

### ④ Voice Fingerprint 확인

생성 전 Spigen 문체 기준을 내부적으로 체크. 출력하지 않음.

- 헤더: 명사형 또는 주장형, 10자 이내 권장
  - O: "패키지 리뉴얼 방향" / "출하량 30% 감소"
  - X: "패키지 리뉴얼 방향에 대한 검토" / "현황 분석 및 방향성 제시"
- bullet: 1줄 이내, 슬라이드당 3줄 이하
- 수치가 있으면 수치 먼저, 설명은 다음
- AI 클리셰 없음: 혁신적인·원활한·극대화·시너지·솔루션·최적화 → 구체 수치·사실로 대체

## Step 2. 슬라이드 구성 제안

### 출력 형식

```
=== 슬라이드 구성 ===

[1] 표지 — (제목)
[2] (슬라이드 제목) — (핵심 내용 1줄)
[3] ...

승인하시면 생성을 시작합니다.
```

### 슬라이드 유형 선택

Step 1.5 ②에서 결정한 **역할**을 그대로 가져온다. 역할 없이 컴포넌트를 고르지 않는다.

| 역할 | 함수 |
|------|------|
| 표지 | `cover()` |
| 설명 / 순서 / 체크 / 결정 유도 | `slide()` |
| 비교 (Before/After, A vs B) | `two_col()` |

### 슬라이드 수 기준

| 목적 | 권장 장수 |
|------|----------|
| 빠른 보고 | 4~6장 |
| 교육·안내 | 6~10장 |
| 제안서 | 5~8장 |

### Step 3 진입 조건

- [ ] 사용자가 구성을 승인했는가
- [ ] 모든 슬라이드 내용이 사용자 제공 데이터에 근거하는가

하나라도 No → 구성 수정 후 재승인.

---

## 수정 요청 처리

**구성 단계 수정**: 해당 슬라이드만 수정 → 전체 구성 재출력 → 승인  
**생성 후 수정**: 해당 슬라이드만 재생성  
**생성 후 구조 변경**: Step 2부터 재시작
