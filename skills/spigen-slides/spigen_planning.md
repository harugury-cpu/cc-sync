# spigen-slides Planning Guide (v4)

## 역할 분리

| 주체 | 역할 |
|------|------|
| 사용자 | 내용 제공, 아웃라인 승인 |
| AI | 내용 구조화, 컴포넌트 선택, 디자인 적용 |

AI는 내용을 창작하지 않는다. 사용자가 준 내용을 구조화할 뿐이다.

---

## Step 1. 내용 수집

### 경로 A: 원본 데이터 있음

문서, 보드 데이터, 채팅 기록, 노트 등 원본이 있으면:
1. 원본을 읽는다
2. 핵심 사실(fact)만 추출한다
3. 추출 결과를 사용자에게 확인받는다

추출 원칙:
- 원본에 없는 내용을 추가하지 않는다
- 수치, 고유명사, 시스템명은 원본 그대로 사용한다
- 요약할 때 의미가 달라지면 원본 문장을 그대로 쓴다

### 경로 B: 사용자 직접 제공

아래 4가지를 묻는다. 이것만 묻는다.

```
Q1. 이 PPT의 목적은 무엇인가요?
    보고 / 교육·안내 / 제안 / 홍보 중 선택 또는 직접 설명

Q2. 청중은 누구인가요?
    (예: 영상팀 내부, 경영진, 외부 고객)

Q3. 전달할 핵심 내용을 적어주세요.
    형식 무관 — 글머리, 메모, 흐름 설명, 키워드 나열 모두 가능

Q4. 테마: dark / light
```

내용(Q3)이 충분히 구체적이면 바로 Step 2로 진행한다.

## Step 2. 아웃라인 제안

### 출력 형식

```
=== 슬라이드 아웃라인 ===

[1] 표지
  제목: (제목)
  부제: (부제)
  → mk_cover()

[2] (슬라이드 제목)
  (핵심 내용 1~2줄 — 사용자가 제공한 내용 기반)
  → mk_xxx()
  → 이유: (왜 이 컴포넌트인지 1줄)

...

승인하시면 생성을 시작합니다.
```

### 컴포넌트 선택 기준

내용 구조를 먼저 파악하고 자동 선택한다.

| 내용 구조 | 컴포넌트 | 판단 기준 |
|-----------|---------|----------|
| 순서 있는 단계 3~6개 | `mk_flow_focus()` | 먼저 → 그 다음 → 마지막 흐름 |
| Yes/No 분기 | `mk_decision_tree()` | 조건에 따라 경로가 나뉨 |
| Before / After | `mk_split_cards()` | 현재 상태 vs 변화 후 |
| 독립 항목 3개 병렬 | `mk_3col_cards()` | 순서 없이 나란히 비교 |
| 섹션 구분 | `mk_section_divider()` | 섹션 전환점 |
| 표지 | `mk_cover()` | 항상 첫 슬라이드 |
| KPI/수치 표 | `mk_kpi_status_detail()` | 수치·달성률·측정식 포함 |

### 슬라이드 수 기준

| 목적 | 권장 장수 |
|------|----------|
| 빠른 보고 | 4~6장 |
| 교육·안내 | 6~10장 |
| 제안서 | 6~8장 |

### Step 3 진입 조건

- [ ] 사용자가 아웃라인을 승인했는가
- [ ] 모든 슬라이드 내용이 사용자 제공 데이터에 근거하는가
- [ ] 컴포넌트 선택 이유가 명확한가

하나라도 No → 아웃라인 수정 후 재승인.

---

## 수정 요청 처리

**아웃라인 단계 수정**: 요청 부분만 수정 → 전체 아웃라인 재출력 → 승인  
**생성 후 내용 수정**: `spigen_inplace_update.py`로 해당 슬라이드만 교체  
**생성 후 구조 변경**: Step 2부터 재시작
