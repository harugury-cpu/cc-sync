# spigen-slides v4 — 검수 방식

v4에서 기획자·디자이너·대상 서브에이전트 3종 파이프라인은 사용하지 않는다.

## 검수 흐름 (v4)

```
생성 완료
  └→ spigen_verify.py (수치 자동 검증)
       └→ PASS: 썸네일 조회 → AI 시각 확인
            └→ 이상 없음: 완료 링크 공유
            └→ 문제 발견: spigen_inplace_update.py로 해당 슬라이드 수정
       └→ FAIL: 해당 항목 수정 → 재생성 → verify 재실행
```

## 왜 서브에이전트를 제거했는가

- 기획자·대상 서브에이전트는 실제 업무 맥락 없이 형식만 검수 → 내용 정확성 보장 불가
- 내용 정확성은 **사용자만 알 수 있다** — Step 2 아웃라인 승인이 이 역할을 대신한다
- 서브에이전트 ❌ → 수정 루프가 내용 창작 문제를 형식으로 해결하려 한 것 → 무한 루프 원인

## 남은 자동 검증

`spigen_verify.py`: 캔버스 이탈 / 폰트 / 색상 / 오버플로 수치 체크  
썸네일: AI 직접 시각 확인 (텍스트 클리핑 / 요소 겹침 / 오렌지 과다)
